## v7.7.2 (2026-06-11) — 페이지 전환 로딩 최소화 (대진표·이벤트 로드 캐싱)

### 🔧 개선·변경
- **페이지 전환 시 반복되던 네트워크 로딩 최소화** — `shelf_load`가 내부에서 호출하는 `_supabase_sched_load`(대진표·점수·잠금)와 `_event_meta_load`/`_event_meta_load_all`(이벤트 메타)는 그동안 캐시가 없어 화면을 다시 열 때마다 매번 Supabase 왕복이 발생했음. 단기 캐시 적용(대진표 20초·이벤트 60초)으로 스코어보드·대진표·이벤트 화면을 재방문할 때 캐시 적중 시 네트워크를 생략 → 페이지 전환 로딩 표시 시간을 단축. 추가로 한 화면에서 `shelf_load` + `_bracket_is_locked` 등으로 같은 날짜를 2~3회 중복 조회하던 것이 1회로 합쳐짐.
- **저장·삭제 즉시 캐시 무효화** — `shelf_save`/`shelf_delete`에서 `_supabase_sched_load` 캐시를, `_event_meta_save`에서 이벤트 캐시를 클리어. `_clear_schedule_cache()` 목록에도 세 함수를 추가해, 점수·잠금·이벤트 저장 직후 본인 화면에는 최신값이 즉시 반영됨. (다른 운영진의 동시 편집은 최대 TTL(20·60초)까지 지연될 수 있음 — 소규모 운영 특성상 영향 미미.)

### 📑 구조
- `APP_VERSION` `7.7.1` → `7.7.2` (헤더 docstring 버전도 동일 갱신).

## v7.7.0 (2026-06-11) — 대진표 저장 데이터 무결성·복원 성능·코드 정리

### 🐛 버그 수정
- **대진표 저장 시 데이터 영구 소실 위험 차단** — `_supabase_sched_save()`가 `delete()` 후 `insert()`를 하던 비원자적 구조에서, 삭제는 성공하고 삽입이 실패(네트워크 오류 등)하면 해당 날짜 대진표가 Supabase에서 사라지고, Streamlit Cloud의 휘발성 로컬 캐시(`.tela_data`)까지 초기화되면 **영구 소실**될 수 있었음. 이제 ① 기존 행을 스냅샷 → ② 삭제 → ③ 삽입 순서로 진행하며, ③ 실패 시 스냅샷을 되살려(롤백) 소실을 막는다. (향후 `schedules`에 `(date_key, match_idx)` unique 제약을 추가하면 단일 upsert로 단순화 가능)

### 🔧 개선·변경
- **Supabase→로컬 복원 성능 개선(콜드 스타트)** — 시작 시 자동 복원(`_restore_from_supabase`)과 백업점검 페이지의 수동 복원에서 `shelve.open()`을 매 날짜마다 열고 닫던 것을, 복원 대상이 있을 때 **루프 밖 1회 open**으로 변경. 최대 500건 복원 시 open/close 반복을 제거.

### ♻️ 리팩토링
- `_login_dedup()` 내부의 중복 `import hashlib` 제거(모듈 상단 540행에 이미 존재). 동작 변화 없음.

### 📑 구조
- CHANGELOG 형식 정합성: v6.3.0의 규칙 외 카테고리 `### 🔐 보안`을 허용 5종 카테고리의 `### 🔧 개선·변경`으로 정리(내용 동일, '보안' 취지는 항목 본문에 명시).

## v7.6.9 (2026-06-11) — 이벤트 왕카드 한 줄 최대 3장(공동 그룹 단위)

### 🔧 개선·변경
- **이벤트 기록 왕카드 3장 제한 (수정1):** 포디움(왕카드)을 **한 줄 최대 3장**으로 제한. 공동 순위는 **그룹 단위로 채우되, 추가 시 3장을 넘기는 그룹은 통째 제외**합니다.
  - 예) 공동1위 2명 + 공동3위 2명(또는 4명) → **공동1위 2장만** 표기.
  - 예) 1·2·3위 각 1명 → 3장, 공동1위 3명 → 3장.
  - 첫 그룹이 단독으로 3장을 넘기면(예: 공동1위 4명) 빈 포디움 방지를 위해 앞 3장만 표기.

## v7.6.8 (2026-06-11) — 매치업 등급 반영(7:3)·이벤트 참여자 전원 표기·요일 중복/설정박스 수정

### ✨ 신규
- **매치업 예상에 등급 반영 :** 예상 승률을 **전적 70% : 등급 30%** 로 블렌딩. 등급 1=최상위→강도 5 … 5=입문→강도 1로 환산해 팀 평균 강도 비율을 반영합니다. 두 팀 모두 등급 보유 회원이 있을 때만 적용(없으면 전적만). **등급 수치는 페이지에 표시하지 않습니다(대외비).**

### 🐛 버그 수정
- **대진보관함 요일 중복 :** 키에 이미 들어있는 요일을 다시 붙여 `(목)(목)`처럼 이중 표기되던 문제 수정.
- **이벤트 기록 참여자 누락 :** 점수가 입력되지 않은 참가자도 **대진표 기준으로 전원 0점 표기**. 또한 점수가 하나도 없는 이벤트도 목록에 노출(이벤트 이름·대진표가 있으면).
- **경기결과 이벤트 설정 박스 넘침 :** 이벤트 이름을 전체폭 한 줄로 분리하고 승·패 점수·설정 저장을 아래 한 줄로 재배치 → 버튼이 박스를 넘거나 초록 입력칸이 어긋나던 문제 해결.

### 🔧 개선·변경
- **회원 등록 폼 회원유형 정렬 :** 회원 유형·배우자 입력을 다른 행과 같은 `1:1:1` 3칸 그리드로 통일.

### 📑 구조
- `_member_grade_map()` 헬퍼 추가(이름→등급, 캐시 180s).

### 🔧 개선·변경
- **세션(자동 로그인) 저장소 shelve → Supabase 이전:** 로그인 유지용 세션 토큰을 로컬 `shelve`(SESSION_PATH) 대신 Supabase `sessions` 테이블에 저장하도록 변경. 기존 shelve는 Streamlit Cloud 컨테이너 재시작 시 초기화돼 자동 로그인이 풀리고, 동시 쓰기에 취약(스레드 비안전)했던 문제를 해소. `_session_save`/`_session_load`/`_session_delete`/`_session_cleanup`을 Supabase 기준으로 재작성했고, 모든 함수를 방어적으로 처리(Supabase 일시 장애 시에도 수동 로그인은 정상 동작, 자동 로그인만 일시 미적용 — 앱이 크래시하지 않음). `sessions` 테이블은 RLS 활성(서비스 키만 접근).

### 📑 구조
- 미사용이 된 `SESSION_PATH` 상수 제거(세션 저장이 Supabase로 이전됨).
